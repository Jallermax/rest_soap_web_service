REST API Documentation

/rest/task_key [GET, POST]

  • GET - get task description
  • POST - get key for moodle

/rest/profiles [GET, POST, DELETE]

  • GET - get list of profiles or specific profile by id
    • Parameters:
      • Boolean pretty[optional] (for pretty json print);
      • Long id[optional] (for getting specific profile);
    • Examle: /rest/profiles?id=100001&pretty=true

  • POST - add new profile in json format from request body
    • Example POST json body: {"name":"Pavel","surName":"Sinitcin","secondName":"Alexandrovich","birthday": "Feb 2, 1995 12:00:00 AM","phone":"+792096219035","email":"sample@gmail.com","position":"Senior QA engineer","projectList":[{"name":"Sberbank technologies"},{"name":"VTB24","description": "Описание 2.","startDate": "Mar 01, 2018 12:00:00 AM","endDate": "Mar 15, 2018 12:00:00 AM"}]}

  • DELETE - delete specific profile by id
    • Parameters: Long id

All profiles exist only during current session and linked with your IP! Site will reset state on next connection or in 7 days for rest service and in 1 hour for wsdl service after first connection with current IP.
